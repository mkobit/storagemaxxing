# StorageMaxxing — UI Executive Summary
*For first-draft UI generation. See full PRD and Flows docs for detail.*

---

## What It Is

A browser-only spatial storage planning tool. Users define physical storage spaces (drawers, shelves, cabinets), pick an organizational system (Schaller bins, Gridfinity, Akro-Mils), set constraints on what goes where, and get a live visual layout plus a bill of materials they can export to CSV or Google Sheets.

**No backend. No accounts. No server.** Everything runs in the browser. State saves to localStorage.

---

## The One Core Loop

```
Define a space  →  Pick a system  →  Set constraints  →  See layout + BOM  →  Export
```

For users who don't know what system they want, Options Mode runs first:

```
Define a space  →  See 3–5 system options side by side  →  Pick one  →  Configure  →  Export
```

## Key Screens (5 total)

### 1. Home / Project List
- List of saved projects (localStorage)
- Each project = one physical location (workshop, one room, etc.)
- **[ + New Project ]** and **[ Quick Start ]** as primary CTAs
- Quick Start skips project setup, goes straight to space definition

### 2. Project View
- Cards for each Assembly in the project
- Assembly = one piece of furniture or zone (tool chest, shelving unit, entryway)
- Each card shows: system selected, validity badge (green/amber/red), total cost
- **[ + Add Assembly ]** and **[ Export All BOM ]** in header

### 3. Assembly View ← primary working screen, most time spent here
Split pane:

**Left panel (300px fixed):**
- Drawer/space tabs at the top — one tab per space in the assembly
- Active tab shows: space name (editable inline), dimensions (W × L × Depth), count multiplier
- Below tabs: constraint list for the active space
  - One row per compatible bin: color dot | bin name + dimensions + price | min stepper | max stepper | on/off toggle | hard lock
  - Bulk toggles: "all auto" / "all off" per depth group
- Collapsed sections below: Variety Packs, Custom Bins, Options

**Right panel (flex):**
- System selector in header: [ SCHALLER ▾ ] with dropdown of compatible systems
- Validity badge: ● valid / ⚠ partial / ✕ invalid with brief explanation
- Stat row: utilization % · bins placed · cost · SKUs
- Canvas: top-down 2D view of active drawer/shelf filled with colored bin silhouettes
- Rulers on X and Y axes with inch/fraction labels
- Below canvas: legend (color → bin type)
- View toggle: [ Top Plan ] [ Front Elevation ] [ 3D ] (3D is Phase 1)
- **[ Options Mode ]** button top-right of canvas area
- **[ Export BOM ▾ ]** button — dropdown: Copy text / Download CSV / Open in Google Sheets

### 4. Options Mode (modal or overlay over Assembly View)
- Triggered from Assembly View
- Shows 3–5 strategy cards — one per compatible system
- Each card:
  - System name + variant description
  - Metrics: utilization % · bins · bin area (in²) · bin volume (in³) · cost · SKUs · constraints met · accessibility · installation required
  - Mini canvas preview (top-down thumbnail)
  - **[ View Layout ]** (expands preview) · **[ Select & Configure ]** (commits, returns to Assembly View)
- No ranking, no stars, no "recommended" label — metrics only, user decides
- If multiple spaces (multiple drawers): metrics shown per space AND as aggregate totals
- Cards side by side on desktop, stacked and swipeable on mobile

### 5. BOM View (tab within Assembly View, or full-screen on export preview)
- Table: SKU | Name | Vendor | Depth | Color | Qty | Unit price | Total
- Grouped by vendor for easy ordering
- Summary row: total bins, total cost
- Per-drawer/per-space breakdown collapsible
- Validity caveat shown if layout is partial or invalid
- Pricing disclaimer always visible: "Prices approximate. Confirm before ordering."

## Interaction Principles

**Live reactive.** Layout updates the moment a constraint changes. No "run" or "calculate" button. The constraint change → MaxRects placement → canvas render cycle is synchronous and sub-millisecond for drawer-scale problems.

**Fraction input everywhere.** All dimension fields are text inputs that accept "15 3/4", "16 1/8", decimals, or plain integers. They display in formatted fraction notation (15¾″) when not focused. Select-all on focus.

**Constraints are per-drawer.** Switching drawer tabs changes the constraint panel. Constraints on Drawer 1 have no effect on Drawer 2. The canvas updates to show the active drawer's layout.

**System is per-assembly.** The system selector in the header applies to all spaces in the assembly. Changing it re-runs all spaces. Individual spaces can override (advanced, not prominent in UI).

**Validity is always visible.** The badge (●/⚠/✕) shows in the header at all times. Clicking it opens a panel listing specific failures with plain language explanations.

**Options Mode has no recommendation.** Metrics are shown. Nothing is ranked. No stars, no "best pick" label.

## Constraint Row — Interaction Detail

```
● [2×4×2]  $0.83    [ − 4 + ]    [ − ∞ + ]    [toggle] [🔓]
  color     price      min          max          on/off   hard
```

- **Color dot:** click to cycle through swatches
- **Min stepper:** −/+ integers; 0 means no minimum; shows value in teal (soft) or red (hard)
- **Max stepper:** −/+ integers; ∞ button toggles unlimited auto-fill; shows value in amber
- **Toggle:** on/off switch; off = excluded from this space entirely (opacity 40%)
- **Lock icon:** 🔓 soft (best-effort minimum) / 🔒 hard (must-place, layout invalid if not met)
  - Lock only appears when min > 0
  - Hard minimum turns the min value red

---

## Validity States

| Badge | Color | Meaning |
|---|---|---|
| ● valid | Green | All hard constraints met |
| ⚠ partial | Amber | Soft minimums unmet; hard constraints satisfied |
| ✕ invalid | Red | Hard minimum(s) could not be placed |

Invalid layouts are still shown — user sees what fit and what didn't. Export is available but shows a caveat. The specific failure is listed: "2×4×2: needed 6, placed 4 — space too full."

## BOM Export Formats

**Copy as text:** Formatted plain text block, vendor-grouped, with total. Paste anywhere.

**Download CSV:**
```
Assembly,Space,SKU,Name,Vendor,Depth,Color,Qty,Unit Price,Total
Tool Chest,Drawer 1,0204-2,2×4×2 Red,Schaller,2",Red,34,$0.83,$28.22
```

**Open in Google Sheets:** Client-side Google Sheets API (gapi). No backend. User authorizes once. Sheet created with: Full BOM sheet + one sheet per assembly + summary sheet.

## Phase 0 Scope (what ships first)

- Schaller Corp catalog only (40 SKUs, 1"/2"/3" depth)
- Drawer spaces only (top-down access, 2D packing)
- Configure Mode only (Options Mode is Phase 1)
- Multi-drawer support with independent constraints
- Drawer count multiplier (×N identical drawers → BOM quantity scaled)
- Validity model (valid / partial / invalid)
- BOM export (all 3 formats)
- localStorage persistence
- Fraction dimension input

**Not in Phase 0:** Shelf bays, 3D view, Options Mode, Gridfinity, Akro-Mils, sketch modeler, project-level BOM, STEP import.

## Design Constraints

- **Dark theme.** Dense information, tool-like. Not a consumer lifestyle product.
- **Compact.** The constraint list may have 15–30 rows. Row height must be small (~32px). Scrollable panel, not paginated.
- **No scroll on the canvas.** Canvas fits the viewport; zoom handled in-canvas.
- **Desktop-first.** Mobile: define space + view options + export only. Full constraint editing is desktop.
- **No animations on constraint changes.** The layout updates instantly; animation would feel laggy.
- **Typography:** monospace for all dimensions and numbers. Sans-serif for labels and body.

## Reference: Existing Prototype

A working React artifact already exists implementing most of Phase 0:
- Schaller catalog, MaxRects packing, constraint system, multi-drawer, BOM
- Split-pane layout: constraint panel left, canvas right
- Drawer tabs, fraction input, rulers, empty-space hatch texture
