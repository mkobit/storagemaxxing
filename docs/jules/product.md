# Product Guidelines: Storagemaxxing Philosophy

## Core Mission
To help users discover what's possible in their storage spaces by providing side-by-side comparisons of different organizational systems, backed by rigorous geometric validation and real-world catalog data.

## Fundamental Principles

### 1. Options-First UX
The tool should never start with a blank canvas or a single system selection without context. The primary entry point is "Describe your space → See possibilities."

### 2. Access Orientation as a First-Class Citizen
We don't just pack boxes. We pack for *utility*.
- **Top-access (Drawers):** Full footprint utilization.
- **Front-access (Shelves):** One-row rule enforced by default.
Always prioritize accessibility over theoretical maximum density.

### 3. Systematic Consistency (The System-Per-Layout Rule)
Options Mode generates layouts using one coherent system at a time (e.g., All Schaller, All Gridfinity). Mixing systems is a power-user feature for "Configure Mode," not a default suggestion.

### 4. Integrity of Fit (Tolerances Matter)
Nominal dimensions are for humans; actual dimensions + tolerances are for the packer. If a layout shows a fit, it must physically work in the real world.

### 5. Validity Over "Optimal"
The tool's job is to prove *validity* first (meeting hard constraints) and *optimization* second (filling remaining space). An "optimal" layout that fails a hard user minimum is a failure.

## Key Product Areas
- **Drawer Filling:** Optimizing for 2D floor footprint.
- **Bin Filling:** Optimizing for 3D sub-division using dividers (e.g., NeoGrid).
- **Shelf Management:** Handling 2.5D stacking and depth constraints.

## User Personas to Keep in Mind
- **The Workshop Owner:** High volume, high precision, SKU efficiency.
- **The Home Organizer:** Aesthetic value, discovery of options, non-standard geometry.
- **The Maker:** Custom systems (Gridfinity/NeoGrid), 3D printing, material cost sensitivity.
